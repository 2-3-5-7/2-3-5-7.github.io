配置备份

optionally-loaded 的插件需要把 start 文件夹替换为 opt，相关 vim-packages、Using git-submodules to version-control Vim plugins

修改别人的插件

  1. 在 submodule 中新建 branch，并添加自己的 remote。目前使用的是和 vim 配置备份相同的 repo，分支名称以插件名称命名

  2. 基于这个 branch 修改，并 push -u

  3. 更新插件的 url 和 branch 为自己的，这样 git push 就可以将插件修改推送到自己的 repo 来备份

  4. 要更新远端的修改，就要切换到原来的 branch pull，然后再把这个 branch merge 到自己修改的 branch 中，然后 push 到自己的 repo。更新了插件需要分别在 submodule 和 主目录提交

    也可以。git push --recurse-submodules=check，git push --recurse-submodules=on-demand

操作技巧

  1. * 查找指针所在单词的下一个位置，# 查找上一个，配合高亮开关使用
  2. 移除行尾空格 :%s/\s\+$//e，Search and replace
  3. wbef 在行中定位，也有大写版本，0^$
  4. m + 小写字母 buffer 中 mark，大写字母全局 mark。’ 或 ` 加字母跳转到 mark，’’ 恢复到跳转前（只能 buffer 内用）
  5. % 转到匹配的条目
  6. 滚动首字母 up、down、forward、back、Home、Middle、Last Quick Movement，zz 将光标内容放入总间，zt、zb
  7. c-r 是重做，redo
  8. c-d、c-t 调整缩进，插入真正的 tab <c-v> <tab> ，Insert tabs in INSERT mode when expandtab is set
  9. 获取某项设置的值加 ?，如 set list?
  10. 重新载入配置比如 vimrc，:so %，How do you reload your .vimrc file without restarting vim?
  11. 操作字符，yl、r、x
  12. 在 change list 中跳转，'. g; g,，How to go back to lines edited before the last one in Vim?
  13. 输入模式 c-p c-n word completion
  14. 5>>.. 缩进，= 自动调整 Shifting blocks visually
  15. https://www.google.com/search?hl=en&as_q=vim+site%3Aspringest.io&newwindow=1
