部署

  1. 安装 git、node.js，参考 hexo 官网，windows、linux 有区别。git 设置 sock5 代理，见 git

  2. github 创建 username.github.io 仓库，实测要用官方的流程设置一下主题才能开通。添加 ssh 密钥

  3. 安装 hexo 并部署

  4. 安装主题和设置主题，可通过查看 star 排名来选择

使用

  • 资源中 json、css、html、yaml 后缀会被解析为网页，统一将后缀中的 . 改成 _

  • 资源文件夹隐藏（只隐藏文件夹本身），避免被 typora 显示

  • 文章存在图片以外的资源时，添加两个资源文件夹链接，需要引用资源文件夹时，本地直接新建同名链接即可

  • 网络中引用资源文件夹要类似 https://github.com/2-3-5-7/2-3-5-7.github.io/blob/main/环境与工具/代码分析

  • 表格前要有空行，且不能有缩进

  • 多个图片放在一行中显示会有问题

  • 创建新建文件模版，.md 类型不支持，所以只能用 .xml，然后重命名来实现

  • Front-matter 模版以 blog.xml 保存到 c:\Windows\ShellNew，在 regedit 新建 计算机\HKEY_CLASSES_ROOT\.xml\ShellNew 项，在项中新建字符串 FileName ，值 blog.xml

维护

  • 升级，参考 将 Hexo 升级到 v4.2.1 中的评论使用 npm-check-updates 在 cmd 中升级。node 本身升级在 windows 下覆盖按照即可
  • 备份
    • 编写 gitignore，忽略 hexo 生成的内容、原始 md 文档、node 包。其中 _post 建立软链接，例如 mklink /D _posts ..\..\Dropbox\blog
    • 以 private 仓库的形式保存到 github，记录对 blog 的修改
  • 还原
    1. 安装 node、git
    2. git clone 分支，然后 npm install
    3. npm install -g hexo-cli
    4. mklink /D _posts ....\Dropbox\blog
    5. 要保存网站更新历史的话 git clone 网站到 .deploy_git，因为每次部署都是 git push -f 会覆盖历史，参考 hexo-deployer-git
