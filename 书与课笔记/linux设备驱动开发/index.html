linux设备驱动开发读书笔记

0. 前言

通过实践领悟代码背后的原理，做技术拼的是傻劲，需要不断重构自己的知识体系，耐心的学习与思考问题。学会思考的方法更重要，打好根基才能游刃有余，需要读者自己举一反三的实践，终生学习，克服一个又一个挑战，培养极强的学习能力，学得越多学得越快，

1. 概述

驱动是软件与硬件的桥梁，如有操作系统则驱动变成操作 系统与硬件之间的桥梁，操作系统通过给驱动制造麻烦从而让对用户的接口统一

  • 字符设备。串行顺序访问
  • 块设备。随机访问，以块为单位
  • 网络设备。用 socket

驱动学习重点、难点

  • 硬件基础
  • C 基础
  • 内核中与驱动相关的部分
  • 并发与同步技术
  • 学习时点面结合，放在整体架构上考虑

代码除了可以在 QEMU 模拟的环境运行，部分还可以直接在 ubuntu 上运行
