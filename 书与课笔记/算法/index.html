排序

  • 选择排序，每次排定最小的那个元素
    • 运行时间与输入无关，对于已经排序好的数组可能是个缺点
    • 数据移动最少，线性
  • 插入排序，对部分有序的情况有优势，是选择排序的两倍
  • 希尔排序，适合没有排序 API 的嵌入式系统
  • 归并排序，NlogN，需要额外空间。链表，自底向上方式，不需要额外内存
  • 快速排序，实现要小心，避免平方级别。切分

查找

键与值，两种基本实现，三种重要数据类型：二叉查找树、红黑树、散列表

需求与问题，大量的查找与插入操作，查找多于插入

链表的无序实现，数组的有序实现，有改进，但当插入和查找是混合的，且表很大时，性能不够

BST，好实现，性能关键取决于 key 的随机性，顺序和逆序插入性能最差。左侧子树小，右侧子树大，能保持顺序

先实现 deleteMin 然后实现 delete 任意节点，关键是对于两个 child 的节点如何处理

平衡查找树

3 links and 2 keys, 2-3 search trees, we continue up the tree, splitting 4-nodes and inserting their middle keys in their parents until reaching a 2-node, which we replace with a 3-node that does not need to be further split, or until reaching a 3-node at the root

为了好实现 2-3 tree 使用 red-black BST，使用 2 个 2-node 表示 3-node，看实现代码来复习

Hash tables

  • 正整数 modular hash 选取质数才能利用上 key 的所有 bit
  • 浮点，用二进制表示的所有位来 hash 而非乘以一个整数
  • 字符串，当作整数取模
  • 复合 key，像字符串一样拼接计算

注意，要有数据实际测试 hash 函数的值是否均匀分布

  • 不支持顺序操作
  • hash 函数的选择，在计算 hash 值时所有 bit 都被利用上，否则会导致性能问题
  • 时间与空间的平衡
  • hash 值缓存到变量中避免重复计算

数组实现的 hash 表要求小于 1/2 ，否则影响性能

应用

对比表格，一般用 Hash，但对于字符串 key、需要顺序操作等情况用红黑 BST

对于稀疏矩阵点乘，hash 表更节省时间、空间

无向无权重

  • 算法实现与图的表示分离，即两个 class
  • DFS
    • 确定是否存在到某一顶点的路径，已经找到它
    • 查找有几个 Connected components，与 union-find 比，union-find 可以边添加边查询
  • BFS 确定是否存在到某一顶点的最短路径
  • V 与 E 不明确给出，V 是字符串

有向无权重

  • 可达性、环不易看出，DAG directed acyclic graph 无环图
  • 拓扑排序，所有的边向下指
    • preorder，调用前
    • postorder，调用后存到队列
    • reverse postorder，调用后存到栈。证明这种就是拓扑序，对于任意 v-> w，w 肯定先入栈。执行 dfs(v) 时，要么 w 被标记了，无环时（有环时有 w->v 的情况），肯定已经返回入栈了，要么没被标记，dfs(w) 先返回先入栈
  • Strong components，关键是有环连接所有顶点。算法，以 GR 的 reverse postorder 用 dfs 遍历 G，每次 DFS 是一个 Strong component
  • Kosaraju 算法证明，先证明连通的顶点通过 dfs 都能遍历到，通过假设 s->v 遍历不到来反证
  • 然后证明 dfs 遍历到的顶点都是连通的，对于 s->v，v 为任意顶点，还要证明在 G 中有 v->s 即 s->v 在 GR 中。因为在 reverse postorder GR 的结果中 s 先于 v （如 v 先于 s，则不会有 s->v，那样 v 就先被 mark 了），说明在 dfs GR 时 dfs(v) 先于 dfs(s) 结束，会有两种情况：dfs(v)->dfs(v) 结束 ->dfs(s) -> dfs(s) 结束，或 dfs(s) -> dfs(v) -> dfs(v) 结束 -> dfs(s) 结束。因为在 GR 中有 v->s （因为 G 中有 s->v），则不可能是第一种情况，只剩第二种，第二种情况表明在 GR 中存在 s->v
  • transitive closure 是将 v->w 的路径用边直连构成的图，因为边密集一般用矩阵表示。查询任意 v->w 的算法待完善，目前就是将 V 个顶点 dfs 的结果保存到数组

无向有权重

  • Minimum spanning tree MST，包含所有顶点的最小权重无环图，权重可以为 0 或负数
  • 一个 cut 将顶点分成两部分，crossing edge 就是其中一个顶点在这部分，另一个在另一部分
  • 最小权重的 crossing edge 一定在 MST 上，反证法假设不在 MST 上
  • 利用上面特性，每次将已在 MST 上的划为一组，V - 1 次后就有 MST 了
  • 数据表示中新增 Edge 类，基于此构建 EdgeWeightedGraph
  • Lazy 算法，维护 crossing edge 队列，每次出队最小权重的，检查这个边的两个顶点是否已经在 MST 上，不符合则加到 MST，并添加邻边到队列，添加邻边时同样检查是否已经在 MST 上了
  • eager 算法，维护每个 V 到 MST 的最短边和对应权重，每次 visit 标记顶点，即将其加入 MST，这时就要更新受到影响的边，如果边的顶点都在 MST 则跳过，否则更新后加入队列，每次出队最小权重的顶点来 visit。个人认为，这个算法的正确性仍然是基于分割的，队列中永远是 crossing edge，出队的就是最小的那个，MST 中的顶点不会被加进去，将要成为 MST 的也会先出队。这里用 marked 来表示在 MST 上的顶点，edgeTo、distTo 记录 crossing edge 和 MST
  • Kruskal 算法，从最小的边开始，只要不与其它树构成环，就是 MST 上的边，从森林转变成树的算法

有向有权重

  • 查找最短路径，Shortest-paths tree SPT
  • 放松 v->w，distTo[w] > distTo[v] + e.weight (v->w weight)，更新 edgeTo[w] = e, distTo[w] = distTo[v] + e.weight。算法中用到的都是放松某个顶点，即放松所有 G.adj(v) 的边
  • shortest-paths optimality conditions，必要性证明：最短路径 distTo[w] 必须满足 distTo[w] <= distTo[v] + e.weight()
    反证法，假设有 distTo[w] > distTo[v] + e.weight()，则 distTo[w] 就不是最短的了
    充分性证明：满足 distTo[w] <= distTo[v] + e.weight()，distTo[w] 就是最短路径了
    不等式消除证明，具体见 P650，注意这里 distTo[w] 指的是某条路径的距离，e.weight 指的是最短路径中的那条边的权重
  • relax 所有边直到没有边可以 relex，因此顺序很重要
  • Dijkstra 算法每次取出最小的 distTo[v] 来 relax，因此 distTo[v] 不会被 relex 为更小，一直不变，而 distTo[w] 只会被 relex 为更小，所以 distTo[w] <= distTo[v] + e.weight() 一直成立。另一种看法是，distTo 按照由小到大的顺序依次被求出
  • 对于无向图的最短路径，可转换成两条反方向同权重的有向边来求得
  • 对于求 s->t 的最短路径，可在 Dijkstra 求到 t 后停止继续求值
  • 无环图按拓扑排序来 relex，distTo[w] <= distTo[v] + e.weight() 一直成立，因为 distTo[v] 在被 relex 后，不会再变，因为不会再有边指向 v，distTo[w] 只会递减
    • 线性的耗时，比 Dijkstra 快
    • 支持负权重边
    • 求最长边，可把权重加负号
  • 并行工作安排的时间取决于最长边，定义 s 和 t ，求 s 到 t 的最长路径。对于要求 v 必须在 w 开始后 d 之前开始的，转换为 v->w 的 -d 边。个人没完全理解，但是根据 P666，w 最多早于 v d 个时间单位，“早”用负边表示
  • 有环有负边的情况，s->v 的路径上有负权重的环，则不会有最短路径。这里实现负环检测，和无负环下的最短路径（有负环下存在简单路径，但该问题太难不学）。Bellman-Ford 算法证明，“算法导论”证明，假设存在最短路径 v1->v2->v3，则每次循环一定会按顺序 relax 一条边，这次循环 relax v1->v2 下次 v2->v3，虽然 v1->v2 也可能多 relax 其它边，但无法保证顺序，不考虑
  • 优化，根据 relax，只有上一轮 distTo 变了的顶点，它的射出边顶点 distTo 可能在这一轮改变，因此用队列记录这些顶点，而不用内层循环遍历所有边
  • 套汇问题，利用对数取负，转换为求最小的负环

好的学习方法是两点，抓住主干，快速迭代，即获得快速高质量的反馈。遇到困难两种处理方法，要么逐步分解到简单步骤，要么绕过先解决其它问题
