1. 必要知识

1.1 ARM

ARM 3 个含义

  • 公司，不生产芯片
  • 一类处理器
  • 一种处理器架构

ARM 处理器家族

  • 经典，ARM 7、9、11
  • Cortex-M，单片机，偏控制
  • Cortex-A，跑重量级操作系统，高性能，偏处理
  • Cortex-R，实时，汽车制动

ARM 处理器架构，包含指令集与编程模型（基本数据类型、工作模型、寄存器组），同种体系结构有多种处理器

  • V1 - V3 废弃，V4 - V6 经典处理器
  • V7 ，支持 Thumb-2 32 位指令集。ARMv6-M v6 与 v7 结合产生的版本
  • V8 兼容 V7，支持 64 位

指令集

  • CISC，指令长度不定，执行需要多个周期

  • RISC，长度通常固定，大多一个周期。ARM

SOC（片上系统），包含 CPU、存储器、外设

1.2 Cortex-M0

低功耗和混合信号的处理，ARMv6-M 处理器架构

两种模式，区别操作的寄存器不一样

  • 线程模式，芯片复位后的模式，执行用户程序

  • 处理模式，发生异常或中断后的模式，处理后返回线程模式

两种状态

  • Thumb状态 ，正常运行时

  • 调试状态，调试时

寄存器

  • R0 - R12，13个通用寄存器，R0-R7 16位或32位，R8-R12 32位

  • R13，SP，两个堆栈指针，MSP 主栈指针，PSP 进程栈指针。处理模式只能用 MSP，线程模式都可用，通过 CONTROL 配置，默认 MSP

  • R14，LR，返回地址

  • R15，PC，下一条指令

xPSR，程序状态寄存器，A 应用 I 中断 E 执行

  • APSR，指令执行后的标志

  • IPSR，异常编号

  • EPSR，Thumb 状态位

PRIMASK 中断屏蔽

CONTROL 选择线程模式使用的 SP，1 使用 PSP， 0 使用 MSP

32 个外部中断 IRQ，1 个不可屏蔽中断 NMI

内部的中断称为异常

指令集，Cortex-M0只支持Thumb指令

  • ARM 指令集，长度 32 位

  • Thumb 指令集，长度 16 位。Thumb-2 技术添加了一些 4 字节指令

1.3 STM32F0

STM32F0 系统架构

  • 二个主模块

    • Cortex-M0 内核及先进高性能总线 (AHB bus)
    • 通用 DMA ( GP-DMA – general-purpose DMA)

  • 四个从模块

    • 内部 FLASH
    • 内部SRAM
    • 专门用于连接 GPIO 口的 AHB2
    • AHB 到 APB 的桥 , 所有的外设都挂在 APB 总线上

4. GPIO

GPIO 口可配置输入、输出，也可复用为其它功能，AD、串口

  • 4 个配置寄存器。MODER、OTYPER、OSPEEDR、PUPDR
  • 数据寄存器。IDR、ODR
  • 置位、复位寄存器。BSRR
  • 复用功能配置寄存器。只是用 GPIO 功能不用配。AFRH、AFRL

  • MODER 2位控制 1 个管脚，16 个管脚为 1 组。2 位表示，输入、输出、复用、模拟，模拟表示输入或输出模拟信号，用于 ADC、DAC。

  • OTYPER 输出类型，推挽、开漏输出。推挽 P、N MOS 都工作，P 输出高电平，N 输出低电平。开漏输出只有 N MOS 工作，只能输出低电平，要输出高电平需外接上拉电阻。

  • OSPEEDR 输出时可以高低电平切换，而这种方波的频率不能超过寄存器配置的频率。设置的频率高了后功耗也会增加。

  • PUPDR 内部有上拉和下拉电阻，不配置则不起作用，配置后相当于设置了管脚默认是输出高电平还是低电平。

  • IDR 工作为输入时的高低电平值

  • ODR 工作为输出时的高低电平值

  • BSRR 用来置位或复位（写0或1） ODR 某一位，好处是不影响 ODR 的其它位，因为可以节省一个读取操作

  • AFRH、AFRL 配置复用功能，4 位对应 1 个管脚

5. 串口

同步通信，通信双方使用同一时钟，SPI、I2C

异步通信，由于没有时钟信号，双方的时钟只能尽量保持一致，通信速率不能太快，UART

串行通信，同一时刻只能 1 bit

并行通信，多根数据线，速度快，占用引脚资源多

单工：只能收或只能发；半双工：可以收发，但不能同时；全双工：可以同时收发

一般来说一根数据线半双工，两根全双工，有时钟线同步，无异步

  • UART 异步全双工
  • SPI 同步全双工
  • I2C 同步半双工
  • 单总线 异步半双工

USART 比 UART 多了同步时钟线的支持，STM32F051K8 有两个串口

串口通信协议（能背诵）

  • 起始位：拉低

  • 数据位5-8：低位（bit0）先发，

  • 奇偶校验位：加上校验位后 1 的个数为奇数个或偶数个

  • 停止位1-2：高

  • 波特率：每秒传送的 bit 位个数，bps，实际没有那么多，因为还要去除协议的位数

常见配置 8N1

  • CR 控制寄存器：收发中断、数据位、奇偶校验、停止位。
  • BRR：设置波特率
  • ISR 中断和状态寄存器：指示接收完数据、发送完数据。正在发送或接收时，通过移位功能进行每个位的收或发，移位完成后产生对应中断或置状态位
  • TDR、RDR：储存发送和接收的数据。串口收发可以通过 FIFO 也可以通过 TDR、RDR

使用 printf 适用于 IAR 和 keil

6. 中断

如果事件发生的频率不高，那么中断比轮询对 CPU 的利用率高

内核部分的中断叫异常，中断的总管家 NVIC，每个外设还有自己的中断管理。比如 GPIO 自己的中断管理叫 EXTI

中断处理的入口地址称为异常向量或中断向量

数值越小优先级越高。3 个固定优先级，-3、-2、-1，不能配置，分别 Reset、NMI、Hard Fault。

4 个可编程优先级，0-3。优先级高的可以抢占优先级低的，即中断嵌套。同级中断同时发生，中断向量编号小的先被处理。如有更高优先级的中断正在处理，新的外部中断会挂起，挂起的中断记录到寄存器。

M0 支持32个外部中断，15个内部异常。而 F051 用了 28 个外部中断，6个内部异常。

GPIO 口用了 16 个外部中断。6组（PA - PF），每组 16 个管脚，每组相同编号的连到同一个通道选择器上（PA0 PB0 … PF0 对应 EXTI0），也就是 6 个管脚只能选其中 1 个来触发中断。通过配置 SYSCFG 来选。

中断与事件的区别？事件发生后会产生脉冲来唤醒 CPU，而中断发生后会执行中断处理函数

中断与事件图，符号右侧没有凹陷的是与门，有凹陷的是或门

中断编程步骤

  1. 使能时钟
  2. 配置 GPIO 为中断功能。配置 SYSCFG 选择
  3. 设置中断优先级
  4. 使能对应的中断
  5. 实现中断服务程序

7. 时钟与定时器

时钟，脉搏，提供工作时的时间概念。时钟源，心脏，振荡器。

  • RC 振荡器，成本低、不稳定。有些 SoC 内部集成充当晶振的备份

  • 晶体振荡器，稳定，成本稍高

STM32 4个时钟源

  • HSI 高速内部时钟，RC 振荡器，8M
  • HSE 高速外部时钟，接晶振或外部时钟信号，4M-16M
  • LSI 低速内部时钟，RC 振荡器，40K，提供给看门狗，也可用作 RTC
  • LSE 低速外部时钟，接 32.768K 晶振，提供给 RTC。RTC 可以在系统掉电后也可使用电池进行计时。

通过分频或倍频提供时钟给各个器件，具体参考 图 10. 时钟树，配置 RCC 寄存器来实现

在 Cube MX 下启用 RCC 高速时钟，在时钟树上选择 HSE，PLL x6 = 48M

汇编启动代码执行 SystemInit 进行默认 HSI 时钟初始化，main 函数执行 SystemClock_Config 进行时钟配置

7.1 SysTick 定时器

SysTick 定时器，M0 内核部件，定时、计数功能，如果时钟是周期的，那么计数就是计时

24 位，递减计数，减为 0 时触发中断。时钟源不分频时为 HCLK

  • SysTick 控制寄存器，开启计数、中断

  • SysTick 重载寄存器，减为 0 重载的数值

  • SysTick 数值寄存器，当前数值

  • SysTick 校准寄存器

因为操作系统要使用，所以已经初始化好了， Cube MX 不用配置。在 SystemClock_Config 调 HAL_SYSTICK_Config(HAL_RCC_GetHCLKFreq()/1000) 设置重载数值，因此 SysTick 触发中断周期 = 1 / 时钟频率 X (时钟频率 / 1000) = 1 / 1000，1 ms

在 SysTick_Handler 中断处理中增加系统 tick 值和调用户的回调函数

注意对微秒级的需求应使用通用定时器，否则中断频率太高，SysTick 主要给系统使用

HAL_Delay 依赖系统 tick 值，在高优先级中断中使用会导致死锁，且中断中应避免延时函数

7.2 通用定时器

6 个通用定时器，1 个基本定时器，1 个高级定时器

通用定时器

  • 定时计数
  • 输入捕获，获得频率、脉宽
  • 输出比较，输出 PWM

高级定时器，带死区控制和紧急刹车，用于 PWM 控制电机

基本定时器，定时计数、驱动 DAC

计数模式

  • 向上，从 0 递增达到某个值后产生中断
  • 向下，从某个值递减达到 0 后产生中断
  • 中央对齐，先向上，产生中断后再向下，如此往复

7.2.1 定时计数

定时器使用内部时钟时，APB 没有分频的话等于 PCLK

Cube MX 的 TIM2 启用内部时钟（其它部分不用设置），系统时钟使用外部晶振 X6，配置 TIM2 的 PSC 为 48 - 1，使用 48 分频（此时频率1M），计数值为 1000000 - 1，1M 触发一次中断来实现 1s 定时，最后开启对应 NVIC 中断。

代码中从中断向量表往里找，最终调用 HAL_TIM_PeriodElapsedCallback，其它回调函数用于不同通道的输入捕获和输出比较功能。注意在回调函数中判断 if(htim->Instance == TIM2)。最后 main 中调 HAL_TIM_Base_Start_IT 启用定时器

7.2.2 输入捕获

输入捕获可以用来捕获外部事件，比如引脚的电平变化（上升沿，下降沿），并记录下变化的时间，通常可以用来测量外部信号的频率或者电平持续的时间

7.2.3 输出比较

此项功能是用来控制一个输出波形，当计数器与捕获/比较寄存器的内容相同时，输出比较功能做出相应动作，比如电平的翻转。通常用于生产PWM波形

8. ADC

性能指标：量程、分辨率、转换时间

精度高转换时间变长，耗时 = 采样时间 + 转换时间

12 位精度耗时 1us（采样时间 1.5T，精度 12bits 转换时间 12.5T，总耗时 14T，最高工作频率 14M）

  • 精度：6、8、10、12 位
  • 转换电压范围：0-3.6
  • 供电范围：2.4-3.6
  • 19个转换通道：16个外部通道、3个内部通道（温度、外部电池、参考电压）
  • 采样时间可配置

时钟源

  • PCLK 的 2/4 分频，耗时固定，但受系统频率影响
  • 片上 HSI 时钟，保证工作在最高频率（14M），但触发事件与开始转换之间的延迟不确定

转换模式

  • 单通道/多通道，单次/多次转换，前扫描/后扫描（从通道 0 到通道 18 还是相反）。间隔转换，外部事件信号开启转换
  • 左对齐/右对齐，转换后的数据储存在 16位 寄存器的高位部分还是低位部分

触发方式，开始方式

  • 软件设置 ADC_CR 的 ADSTART = 1
  • 外部事件，定时器 TRGO、输入引脚，上升沿、下降沿触发

中断产生方式，对于单通道转换没有区别

  • 在多通道转换时，一个序列转换结束产生中断。ISR 寄存器 EOS 标志
  • 在多通道转换时，一个通道转换结束产生中断。ISR 寄存器 EOC 标志

模拟看门狗

  • 外部电压值超过设定的范围，就会产生中断，进行报警

9. DMA

  • 设备发送 DMA 请求，DMA 搬运数据到存储器

  • 5 个 DMA，可配置优先级，优先级高的获得总线，相同优先级的通道编号小的优先级高

  • DMA 需要源、目的、长度。源与目的可以是存储器或外设，因此会组成 4 种搬移情况。长度包含数据的宽度（8bit、16bit…)和数据的数量，源与目的的数据宽度通常保持相同

  • DMA 增量模式，如将内存配为增量模式，那么从同一个寄存器读每次会存储到下一个内存单元

  • DMA 循环模式，如设置DMA从寄存器到内存搬移 4 次，内存增量模式，开启循环模式后，到达 4 次后 DMA 会继续搬移到内存的起始位置，也就是实现了循环缓冲区。ADC 连续采集与 DMA 循环模式需要相互配合，ADC 不断采集然后发起 DMA 请求，而 DMA 不断的搬运数据。

  • 中断：传输一半、传输完成、传输出错

ADC 多路采集 DMA 实验

  • ADC 上添加 DMA，DMA 中断默认启用，DMA 如果配置了循环模式则 ADC 也选择连续模式。注意避免中断太频繁

  • 开启ADC DMA 使用 HAL_ADC_Start_DMA 第 3 个参数为 DMA 传输的数据数量，第 2 个参数可以强转为 uint32_t *

  • DMA 完成的回调函数在调用 HAL_ADC_Start_DMA 时注册，最终我们需要实现回调 HAL_ADC_ConvCpltCallback

10. I2C

I2C，详细请参考 12.md I2C 子系统部分

  • 同步半双工串行，线与，上拉电阻，默认高电平

  • SCL 高电平时，SDA 变化，拉低表起始，拉高表结束

  • 7 位地址（也有 10 位地址），加上 0 写 / 1 读。接收方拉低进行 ACK （应答）

  • 数据传输时 SCL 高电平时 SDA 不变，SCL 低电平时 SDA 变化。每个字节从高位开始发

  • 对于 读，主设备不想读时发 NACK（维持高电平不拉低）。而且通常读是需要先写地址（要读的地址）然后进行第 2 次开始才是读

MPU6050 实验

  • I2C 可以用 GPIO 模拟或者硬件 I2C
  • Cube MX 开启 I2C，将 SCL、SDA 管脚改对后没有其它需要配置的了
  • 代码里使用 HAL_I2C_Mem_Write 和 HAL_I2C_Mem_Read 来读写寄存器，其中第 4 个参数指定地址长度，只支持 1B 和 2B 寄存器地址，写入或读取的数据长度以字节为单位
