Zigbee

1. Zigbee 介绍

无线通信技术。zigbee、蓝牙、wifi、红外。

zigbee

  1. 短距离。功率大几公里也可以
  2. 低复杂度，容易开发
  3. 低功耗、低成本
  4. 传输数据速率低

三种协议对比，ZigBee 主要用在短距离、少量传输、电池供电系统

zigbee 体系结构

  1. 应用层。APL、ZDO
  2. 应用支持子层。APS
  3. 网络层。NWK
  4. 介质访问层。MAC
  5. 物理层。PHY

1-3 zigbee 标准，4-5 IEEE 802.15.4 规范

协调器 ZC

  • 为整个网络选择 PAN ID。相当于班级号
  • 为每个设备分配网络地址。相当于学号
  • 初始化、终止、转发网络消息

路由器 ZR

  • 允许设备加入网络
  • 扩展网络覆盖范围
  • 数据包的路由

终端设备 ZED

  • 网络数据的采集

2. 协议栈与基本概念

以下概念参见视频 ZigBee Application Profiles, Clusters and Endpoints

视频中 PPT 的 client 、server 中的 in、out 写反了。profile 包含了多个 device type，应该都是标准所定义的设备类型，注意区分视频中提到的 device 是指的 device type 还是物理设备也就是 node。

下面的概念概括起来就是，一个 node 包含了多个 endpoint，每个 endpoint 实现了某个 profile 中的某个 device type，一个 endpoint 包含了多个 cluster，而且 cluster 还分 in/server、out/client。但是 cluster 作为命令与属性集合不只是被 device type 包含的关系，也可能同时存在于多个 profile、device type 当中，如下图。

profile 规范

  • 一个应用领域，如智能家居。16 bit profile ID，由 zigbee 联盟分配
  • profile 包含了不同的 device type，通过 16 bit device ID 区分。device type 又包含了多个 cluster
  • 如果公共 profile 没有适合的，需要自定义一个 profile ，然后向 zigbee 联盟申请 profile ID

cluster 簇

  • 与某个功能相关的消息类型集合。16 bit ID，在 zigbee cluster library(ZCL) 定义
  • 多个 profile 之间的会有相同的功能，所以一个 cluster 可以存在于多个 profile 中
  • 每个 cluster 包含了属性集合和命令集合
  • cluster 被编组成 funtion domains
  • cluster 还被分成 client 和 server，client 发消息给 server，server 包含了属性。比如一个开关作为 client 发送开灯命令给作为 server 的灯，server 维护着属性，灯的状态

下图左侧属于同一 funtion domain 的 cluster 被标成相同颜色，所有的 domain 构成了 ZCL。右侧每个 profile 中包含了多个 cluster，同一 cluster 可以出现在多个 profile 中

endpoint 端口

  • 每个 endpoint 实现了某个 profile 中的某个 device type。是 node 所有支持的功能的一个子集。有人比喻成操作系统（node）中的一个服务（endpoint）或者一个进程所使用的端口号（endpoint）
  • 每个 node 最多 255 个 endpoint。0 给 ZigBee Device Profile：ZDO 设备，用于网络配置和管理，获取设备信息。240-255 预留，255 用于广播
  • 其它的 endpoint 没有标准定义，所以需要通过一些服务发现手段来查询获取

当一个 node 加入网络中后，需要查询其它 node 都有哪些 endpoint，每个 endpoint 提供了什么服务。当查询一个 endpoint 时，会返回如下信息

  • 一个 descriptor 和 一个 cluster list
  • cluster list 包含了支持的 in/server clusters、out/client clusters
  • descriptor 包含了实现的 profile 和 profile 中的 device type
  • client 的输出簇要与 server 的输入簇对应（来自创客学院视频）

3. 组播、串口、地址的分配、获取

组播通信

  • 组播发送时填写的地址与加组时的 group id 是一个概念，要自己定义一个值
  • Tools 下 f8wConfig.cfg 中的 -DRFD_RCVC_ALWAYS_ON 改为 TRUE 后终端节点才会接收组播报文
  • 注意 RFD_RCVC_ALWAYS_ON 开启后，广播也属于组播的一种，终端节点发出的报文自己也会收到一份。发送组播时目的 EP 不起识别作用，因此解决这个问题只能通过每个设备一个 clust id 来屏蔽终端节点收到的自己发出的消息，不过估计 ZED 发组播的情况不多。
  • 测试发现不加组也能给那个组发报文，只是收不到组内报文
