1. bootloader

功能硬件初始化，加载内核

uboot 是 bootloader 一种，免费 省钱，开源 可以深度定制成有竞争力的产品，支持多 开发时间少

nor flash 需要取消写保护，先擦除再写。默认是 1，1变0可以，0变1唯一方法就是擦除

uboot 启动流程

汇编阶段

  • 越简单的越稳定。设置 SVC 模式，关看门狗、中断、mmu、cache

  • 基本硬件初始化，时钟、串口、flash、内存

  • 自搬移 uboot 到内存

  • 准备栈

C 阶段

  • 大部分硬件初始化，GPIO、串口、网口

  • 搬移内核到内存，运行内核

2. 内核

linux 内核不是linux操作系统

  • 免费开源
  • 可以移植，支持的硬件广泛
  • 可裁剪。功能强
  • 不是实时系统。Vxwork

linux 内核子系统

  • 进程管理
  • 内存管理
  • 文件系统
  • 网络协议
  • 设备管理

设备树是 cpu 与 外设的连接方式，这样驱动可以与硬件无关

看链接脚本获得第一句话的位置

公司代码与官方代码用对比软件，快速融入团队。参考类似配置再改

先实现串口输出方便调试

启动流程

  1. 自解压内核 decompess (arch/arm/boot/compressed/head.S)
  2. 运行内核汇编部分 head.S 入口stext (arch/arm/kernel/head.S)
    ​ 检测合法性(CPU 类型,机器类型)
  3. 运行内核C部分 start_kernel (init/main.c)
    ​ CPU,机器参数的安装 setup_arch
    ​ 中断，定时，终端，内存等最基本的初始化
    ​ 创建核心进程 kernel_init 运行,启动多任务调度
  4. 挂载 rootfs
  5. 运行第一个应用程序init (一般是 linuxrc）

调试方法

  1. 点灯

  2. 打印
    puts (内核解压前)
    printascii (console初始化前)
    printk (内核解压后,信息输出显示是在 console 初始化之后)

通过proc在运行时查看和修改日志级别，数字越小优先级越高，改第一个数
cat /proc/sys/kernel/printk 显示 4 4 1 7
echo “7 4 1 7” > /proc/sys/kernel/printk 后
cat /proc/sys/kernel/printk 显示 7 4 1 7

4.19 系统移植

添加驱动

  1. 选择驱动存放目录 (或任意目录）
  2. 改 Makefile，obj-$(CONFIG_LED)
  3. 改 Kconfig（界面可配置），config LED
  4. insmod
  5. 创建设备节点（应用访问驱动的入口），mknod /dev/<device> [c|b] <major> <minor>
  6. 运行测试驱动的应用程序

字符设备、平台设备，平台设备就是把寄存器地址存在设备树文件中，方便维护

常见的文件系统

  • NFS ROOTFS 常用于开发调试，和测试文件系统内容
  • CRAMFS 只读。常用于安全性要求高，需要数据不被改写情况。（常为了安全性，把基本rootfs做为CRAMFS. 再挂载别的文件系统如jffs2等实现可写入数据）
  • JFFS2 因nand nor flash都支持，嵌入式产品中常用。
  • YAFFS2 专门用Nand Flash。 一般手机开发都用它
  • EXT2/EXT3 常用于电脑硬盘
  • RamDisk 把内存抽象为文件系统。 适用于内存较大，而flash较小情况
