1. MQTT 通信搭建

  1. win10 mqttfx 作为一个客户端，参照 emqtt 推荐 的 Eclipse Paho C 在虚拟机下 git clone 并编译作为另一个客户端，下载win10 稳定版 emqtt 服务器 ，注意解压缩的路径不要有空格
  2. 启动服务器第一次是需要 Ctrl+c 然后选择 n 才会启动。通过本机的 18083 端口访问管理页面，用户名密码 admin、public。通过打印得知监听的 tcp 端口为 1883。配置客户端时使用
  3. 启动用来订阅的 windows 客户端，添加 profile 并连接服务器，订阅 topic “MQTT Examples”。修改 linux 下的 paho.mqtt.c\src\samples\MQTTClient_publish.c ADDRESS 为服务器地址，编译并运行，注意需要设置库路径 export LD_LIBRARY_PATH
  4. 用 wireshark 抓端口 1883 包，由于 vmware 虚拟机（好像 virtual box 没问题）的缘故，双向都会抓到两份包。使用 wireshark 目录下的命令行工具 editcap -d old.pcapng new.pcapng 移除重复的包

2. 阿里云物联网

总体开发流程

设备端开发流程

物模型，简称TSL，即Thing Specification Language

功能类型 说明
属性（Property） 一般用于描述设备运行时的状态，如环境监测设备所读取的当前环境温度等。属性支持 GET 和 SET 请求方式。应用系统可发起对属性的读取和设置请求。
服务（Service） 设备可被外部调用的能力或方法，可设置输入参数和输出参数。相比于属性，服务可通过一条指令实现更复杂的业务逻辑，如执行某项特定的任务。
事件（Event） 设备运行时的事件。事件一般包含需要被外部感知和处理的通知信息，可包含多个输出参数。如，某项任务完成的信息，或者设备发生故障或告警时的温度等，事件可以被订阅和推送。

2.1.2 移植

SDK 结构

乐鑫基于 8266 最新 SDK 已经进行了移植，没有验证过，以后可以参考 乐鑫移植

移植步骤

  1. SDK功能配置
  2. 进行HAL函数的实现
  3. 调用SDK提供的API进行产品功能实现
  4. 对SDK进行编译

移植方法综合 SDK 自带编译法 和 代码抽取法

  1. make menuconfig 选择功能，包括调试打印的级别，make reconfig 选择 8266 平台，生成 libiot_sdk.a
  2. 实现 output/eng/wrappers/wrapper.c HAL 函数，hal_printf 要使用 printf 才能打印到串口，vprintf 不好用。移植例子可以参考移植好的 linux 例子
  3. 修改 8266 的 makefile 使 libiot_sdk.a 和 wrapper.c 编译到工程中，还有 include 的多个文件放入 8266 工程的 include 文件夹
  4. 参照例子代码实现设备与云平台的通信
