1. LoRa 概述

优点

  • 远距离
  • 低功耗
  • 低成本
  • 免牌照频段，不给运营商交钱

LoRa 三个参数影响了通信速率、耗电（电池寿命）、通信距离。详细计算可以通过 LoRa Modem Calculator Tool

  • 带宽 BW，越大速率越高，但降低了接收灵敏度
  • 扩频因子 SF，越大信号质量越强，速率下降
  • 编码率 CR，提高数据可靠性

4 个概念参考 chips-chirps-symbols-and-bits，详细的概念介绍 [LoRa 重要知识](LoRa 重要知识.pdf)，参考资料中的教学视频也很好

  • 注意符号 symbol 和码片 chip 与我们经常用到的 bit 不同。换算关系为 1 个 symbol 对应 2^SF 个 chip，SF 个 bit

  • 发送一个符号 symbol 的时间

    比特率

    算上 CR 后

  • 芯片手册包含了根据报文大小计算传输时间的公式，但其实用前面的 LoRa Modem Calculator Tool 工具就可以算出

LoRaWan 终端分为三个 class，实时性越来越好，功耗越来越高。参考 lora 联盟发布的 what-is-lorawan

  • A 设备上报时间随机，下发的控制信息只能跟随上报时间后的两个固定时间窗口
  • B 在 A 功能基础上，增加时间同步 beacon，告诉终端何时开启接收
  • C 除了上报报文的时候，其它时间都开启接收

2. LoRa 固件移植

官方固件结构，推荐读 doc/README

  • 应用层 main.c
  • 无线驱动层 radio
  • 硬件抽象层 sx12xxEiger

硬件连接MCU 与 LoRa，DIO0 -DIO5、SPI、RESET

移植需要做的

  • 不支持 M0，改成包含 m0 的头文件
  • 固件库改成 HAL 库，注释掉 init 相关函数，修改 GPIO 相关和 GET_TICK_COUNT 宏为 HAL 接口，实现 SpiInOut 函数
  • IO 口不同，LED IO定义，HAL IO 定义，没有 reset 和 rxtx 设置连接

3. PingPang 与 串口透传

修改 LoRaSettings 两个参数

  • RFFrequency 改为 492000000，sx1278 频率范围137-525MHz
  • RxSingleOn 改为 0，连续接收

注意

  • 连续接收模式不会产生 TIMEOUT 返回值，收到包后也会保持在 RFLR_STATE_RX_RUNNING 状态，免去再次调用 Radio->StartRx()，所以需要自己判断。如果改成单次接收，那么会有大量报文接收不到，具体区别见函数 SX1276LoRaProcess 的状态切换和 sx1276_77_78_79.pdf 芯片手册的 LoRa Receive Sequence 图以及使用场景说明
  • 开发板需要在 LCD 和 LoRa 芯片两个 SPI 间切换，所以每次读写 SPI 需要软件操作片选信号。同时 LCD 的 MISO 要改为 GPIO 输出，来表明输出是指令还是数据

串口透传

  • cubemx 打开串口中断，串口 DMA
  • 初始化使能串口空闲中断，开启串口 DMA 接收
  • 在串口中断中取得收到的串口数据长度，轮询串口数据长度变量，然后调用 lora 发送

4. LoRa 自组网

目前自组网通信的设计类似 LoRaWan class A 和 class B，下发的控制数据只有在终端上报后，但终端上报时间却不是随机的，而是基于 slot 的固定周期

实现文档见代码文件夹\net 中的 README.md
