新建工程，写start.s，指定工程的链接脚本

7个工作模式

  • user，非特权
  • IRQ，中断
  • supervisor，超级
  • FIQ，高优先级中断
  • abort，异常
  • undef，未定义指令，浮点运算
  • system，user 模式的特权模式

37个寄存器

  • PC、CPSR（当前工作模式 current program status register）、5个 SPSR（保存的 saved program status register）
  • 30个通用寄存器，r13 SP，r14 LR，r15 PC
  • ARM 状态指令 4B，thumb 状态指令 2B
  • r13 - r14 每个模式独立的
  • r8 - r12 对于 FIQ 也是独立的
  • CPSR（32位） 0-4 模式，28-31 进位、为0等标记位，还有中断禁止位。读CPSR mrs，写 CPSR msr，swi 触发软中断

mov指令中的立即数限制，ARMv5 指令中只有12位可以储存立即数，其中8位储存数（范围0-255），另外4位表示循环右移的位数，位数只能是0-30之间的偶数

两种方法解决， ldr r7,=#370 从内存中读数据，或者用两条指令运算得出想要的数。ARMv7 可以用 movw 载入16位的数

异常种类

  • 硬中断，IRQ、FIQ
  • 软中断，系统调用，进入内核
  • 复位，比如手机死机
  • 未定义指令
  • 数据异常，数据越界

多个中断源接中断控制器。中断处理完后要清除中断源。0 - 0x1c 为异常向量表

  • 复位与软中断，对应 SVC 模式

  • prefetch、data abort，对应 abort 模式

  • user 与 system 模式不是异常触发切换的，是程序员修改 CPSR 切换的

中断处理实现

  • 定义异常向量表

  • 定义栈，保护与恢复现场

  • 获取软中断号

串行通信

  • uart，双线，全双工、异步。加校验位有奇数个 1 就是奇校验，偶数个 1 就是偶校验
  • i2c，双线，半双工、同步。有时钟线减少出错
  • spi，三线，全双工、同步。速度最快

设置功能模块

uart：开始位拉低，发送 8 bit 数据，发送停止位拉高

I2C：一个主多个从，可以主从切换，主发送，从接收

开始位拉低，7位从地址，rw位（1表示读），应答。传数据1B，应答。停止位

接收应答时，主设备从输出切换到输入，电平恢复高，从设备负责拉低进行应答

约定先传高位或低位，上升沿保持，下降沿改变，或者反过来

中断控制器，3种中断，Shared Peripheral Interrupt (SPI)， port no，id

pwm，电机控制。设定两个值，比如 159 和 100，则 159 递减到 100 时高低电平翻转，继续递减到 0 时再次翻转，然后重新从 159 开始计数。
