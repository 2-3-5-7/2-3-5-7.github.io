2. I2C 子系统

I2C：一个主多个从，主机是指可以发 SCL 的设备。更多参考 I2C总线协议

  • 传输。开始位1b拉低，从地址7b，rw位（1表示读），应答1b。传数据1B（先高位后低位），应答1b。停止位1b拉高
  • 应答。接收应答时，主设备从输出切换到输入，电平恢复高，从设备负责拉低进行应答
  • 时序。高电平保持，低电平改变。但开始位和停止位在保持（高电平）时变化
  • 注意读模式发送时主最后发 NACK 来通知从释放 SDA 从而让主能发停止位，读写模式切换时要重新发设备地址
  • 支持多主，线与逻辑关系实现仲裁和时钟同步。仲裁，发现 SDA 上的电平与自己发的不一致时退出竞争，因此胜出的主机不会丢失数据。时钟同步，低电平时间是最长低电平，高电平时间是最短高电平
  1. i2c_driver 驱动函数。i2c client，设备树，设备信息
  2. i2c core
  3. i2c adapter
  4. i2c algorth

2、3、4 已实现，make menuconfig 选择即可，需要实现最上层的设备驱动和设备树信息

3. 设备树

bootloader 将 dtb 地址告诉给内核，内核读取后展开(flattened)成二叉树。结点(node)和属性(property)，属性里有 name 和value.

方括号 一个字节数据，尖括号 u32

compatible 制造商,型号

address-cells、size-cells 对应子节点占用个数
