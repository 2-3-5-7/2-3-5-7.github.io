RFID

asset、asset-net

1. RFID 概述

物联网，感知层、网络层、应用层。RFID 属于感知层

射频识别，Reader 读卡器，Transponder 卡片 免电池、免接触

读卡器发射电磁波传递能量与时序，卡片（RFID 标签）返回信息

频率划分

  • 125K - 134K 低频。穿透能力强，波段没限制，衰减快、距离短，传输慢。畜牧业、门禁
  • 13.56 M 高频。10 cm，支持防冲撞
  • 860M - 915M 超高频。1m
  • 2.4G - 5G 微波

RFID 标签组成：RFID 晶片、天线、电力来源（有源、无源）

传数据方式，两种耦合。低频、高频使用电感耦合，超高频、微波使用电容耦合，而且通常是有源的

RFID 标签的数据存储。key 密码、access 访问控制

防冲突机制，多张卡片进入读写射频场时，如何选出 1 张

  • 面向比特。多张卡片比较序号大的胜出
  • 面向时隙。读卡器发序号，卡片来选，然后读卡器叫号，被叫的胜出
  • 面向比特、面向时隙混合

2. 高频 RFID

读写操作流程，1 - 3 参考 ISO14443-A 协议，由于协议没有规定读写细节，所以 4-5 参考 S50 手册、例子代码、读写命令字参考 PN532 手册中How to use the PN532 as a Mifare reader 部分。

目标是弄懂这 5 个步骤中芯片发送的数据和卡片返回的数据格式，即协议。芯片使用 RC522

  1. REQA / WUPA 请求
  2. ANTICOLLISION 防冲突，获得卡片 ID
  3. SELECT 选卡
  4. 认证
  5. 读写、增减操作

2.1 ISO14443-A 协议

ISO14443-A 协议，看 4 本中的第 3 本 (part 3) 初始化与防冲突，14443 没有规定具体读写细节

PICC 卡片，PCD 读卡器

3 种帧格式

  • short frame，数据 7 位（即没有 bit7），没有奇偶校验位，LSB 先传

  • standard frame，n字节，每个字节结束有奇校验位，LSB 先传

  • bit oriented anticollision frame，7字节被分成两部分按照 standard frame 传输，part 1 PCD to PICC，part 2 PICC to PCD

命令与帧格式 发送数据说明与返回数据
REQA
short frame		 0x26
返回卡类型 2B，S50 为 0x4 0x0。参见 Mifare 手册 ATQA 值
WUPA
short frame		 0x52
WUPA 可以唤醒 HALT 状态的设备
返回卡类型 2B，S50 为 0x4 0x0。参见 Mifare 手册 ATQA 值
ANTICOLLISION
bit oriented anticollision frame		 0x93 0x20
SEL 1B + NVB 1B
发送后卡片会返回 ID 4B + checksum(BCC) 1B
注意如存在冲突则这个步骤会多次循环，具体见下面冲突检测
SELECT
standard frame		 0x93 0x70 ID1-ID4 checksum(BCC) CRC16
SEL 1B + NVB 1B + ID 4B + checksum(BCC) 1B + CRC 2B
发送后返回 SAK，S50 为 0x08。参见 Mifare 手册 SAK 值
HLTA
standard frame		 0x50 0x00 CRC16
  • SEL 1B，0x93 0x95 0x97 对应 level 1、2、3，对应 uid 4B、7B、10B，我们的卡是 4B，所以 SEL 为 0x93
  • NVB 1B，高 4 位字节数，低 4 位比特数，如 0x24 表示 2个字节加上4位。NVB 的值包含 SEL 与 NVB 本身，不包含 CRC

PICC 状态

  • POWER-OFF。没电
  • IDLE。上电后，等 REQA 或 WAKE-UP
  • READY。收到 REQA 或 WAKE-UP，进行防冲突
  • ACTIVE。被读卡器选中
  • HALT。可通过 WAKE-UP 转换为 READY

完整的冲突检测步骤，更具体的查看手册上这个表后面的流程图，以及附录的 Communication example Type A 的Figure A.1

2.3 mifare one IC S50

恩智浦 NXP

  • 大小 1K，分 16 个区（sector），每个区分 4 块（block），每块 16B
  • 每个 sector 的最后一个 block(block 3) 存放密码 keyA 、keyB、对 sector 进行读写的条件 access bit
  • key A 永远不可读
  • sector 0 block 0 存放 ID 和 厂家信息
  • 读写操作时用的地址从 sector 0 block 0 开始算 0，一直到最后的 sector 15 block 3 算为 63

3 个控制位 C1 C2 C3 不同取值对应的权限查询 S50 手册的表格，最终取得的读写权限是 控制位 与认证时使用的是 keyA 还是 keyB 联合决定的。另外项目文件夹里有这 4个字节的计算工具。

同一个 sector 的读写认证一次后可以连续对这个 sector 进行读写而不用再次认证

修改了密码后，重新请求时就要新密码认证了

3. NFC

3 种工作模式

  • 卡模式，当公交卡，与刷卡机配合
  • 读写器，读写卡片，与卡配合
  • 点对点，类似蓝牙，进行数据传输，与另一个 NFC 终端配合

NFC 与 RFID 区别

  • RFID 的卡与读写器是两个设备，NFC 可以进行信息交互
  • NFC 只是高频段，RFID 多个频段
  • NFC 针对消费的短距离通信，有源 RFID 可以长距离，比如 ETC

NFC 与 蓝牙

  • NFC 距离短，10 cm
  • NFC 传输速度慢
  • 优点是 NFC 的连接建立时间短 0.1s

可以先用 NFC 建立连接，再用蓝牙传输
