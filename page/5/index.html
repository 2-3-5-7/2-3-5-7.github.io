1. 进程

1.1 理论

进程里面除了程序还有 系统数据段

  • 进程控制块 pcb 即 task_struct，包含 pid、进程用户、文件描述符表、进程状态、优先级、进程返回值
  • cpu寄存器值。PC
  • 堆栈

进程类型

  • 交互进程，shell启动，前台运行，后台运行时无法读取终端输入，但可以输出
  • 批处理进程，与终端无关，提交到作业队列中运行
  • 守护进程，与终端无关，直到系统结束才关闭，一直在后台

进程状态

PROCESS STATE 状态 什么时候产生
R running or runnable 运行态/就绪态 不一定占用 cpu，可能没有时间片
S interruptible sleep 等待态，可否被信号打断，可中断 等待事件或资源
D uninterruptible sleep，Disk wait 等待态，可否被信号打断，不可中断 等待事件或资源。因为不响应信号，所以没办法被用户结束
T stopped, traced 停止态，运行被中止 比如 gdb 调试时，或者 ctrl + Z
Z zombie 死亡态，僵尸态 已经终止但 pcb 没有被释放，pcb 里面有返回值

1. MQTT 通信搭建

  1. win10 mqttfx 作为一个客户端，参照 emqtt 推荐 的 Eclipse Paho C 在虚拟机下 git clone 并编译作为另一个客户端，下载win10 稳定版 emqtt 服务器 ，注意解压缩的路径不要有空格
  2. 启动服务器第一次是需要 Ctrl+c 然后选择 n 才会启动。通过本机的 18083 端口访问管理页面，用户名密码 admin、public。通过打印得知监听的 tcp 端口为 1883。配置客户端时使用
  3. 启动用来订阅的 windows 客户端，添加 profile 并连接服务器，订阅 topic “MQTT Examples”。修改 linux 下的 paho.mqtt.c\src\samples\MQTTClient_publish.c ADDRESS 为服务器地址，编译并运行，注意需要设置库路径 export LD_LIBRARY_PATH
  4. 用 wireshark 抓端口 1883 包，由于 vmware 虚拟机（好像 virtual box 没问题）的缘故，双向都会抓到两份包。使用 wireshark 目录下的命令行工具 editcap -d old.pcapng new.pcapng 移除重复的包

1. Zigbee 介绍

无线通信技术。zigbee、蓝牙、wifi、红外。

zigbee

  1. 短距离。功率大几公里也可以
  2. 低复杂度，容易开发
  3. 低功耗、低成本
  4. 传输数据速率低

三种协议对比，ZigBee 主要用在短距离、少量传输、电池供电系统

zigbee 体系结构

  1. 应用层。APL、ZDO
  2. 应用支持子层。APS
  3. 网络层。NWK
  4. 介质访问层。MAC
  5. 物理层。PHY

STM32

asset、asset-net

1. 必要知识

1.1 ARM

ARM 3 个含义

  • 公司，不生产芯片
  • 一类处理器
  • 一种处理器架构

ARM 处理器家族

  • 经典，ARM 7、9、11
  • Cortex-M，单片机，偏控制
  • Cortex-A，跑重量级操作系统，高性能，偏处理
  • Cortex-R，实时，汽车制动

ARM 处理器架构，包含指令集与编程模型（基本数据类型、工作模型、寄存器组），同种体系结构有多种处理器

  • V1 - V3 废弃，V4 - V6 经典处理器
  • V7 ，支持 Thumb-2 32 位指令集。ARMv6-M v6 与 v7 结合产生的版本
  • V8 兼容 V7，支持 64 位

指令集

  • CISC，指令长度不定，执行需要多个周期

  • RISC，长度通常固定，大多一个周期。ARM

SOC（片上系统），包含 CPU、存储器、外设

嵌入式应用程序设计综合教程，第 6 章

1. 分层

  • 应用层。提供服务的应用协议、应用程序，跑在用户态。ftp、http、telnet
  • 传输层。应用程序间的通信，决定送给哪个应用程序。tcp、udp
  • 网络层。通过路由实现主机间通信。ip、icmp、igmp
  • 网络接口层。外设的驱动所在的层，用于屏蔽硬件差异。ppp、arp、rarp

对于 7 层模型，应用层 对应 应用层、表示层、会话层。网络接口层 对应 数据链路层、物理层
